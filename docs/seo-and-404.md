# SEO и 404 для Яндекса

## Что сделано

1. **Title и description в исходном HTML**  
   При сборке для статических маршрутов (/, /about, /services, /contact, /faq, /brands, /blog) генерируются отдельные `index.html` с подставленными `<title>` и `<meta name="description">`. В «Просмотр кода страницы» и для робота Яндекса эти теги уже есть в разметке.

2. **Дефолтные теги в корневом index.html**  
   В шаблоне добавлены теги по умолчанию (главная страница), чтобы даже без JS в исходном HTML было что показывать.

3. **404**  
   Для корректного ответа 404 по несуществующим URL сервер должен отдавать статус 404, а не всегда 200 с одним и тем же HTML.

## Настройка 404 на хостинге

### Netlify

Используется `netlify.toml` в корне проекта:

- Для несуществующих путей Netlify сам отдаёт 404 и показывает `404.html`.
- Для SPA добавлен редирект: `/blog/*` → `/index.html` (статьи блога не пререндерятся, отдаётся один index.html).

Не добавляйте общий fallback `/*` → `/index.html` с кодом 200, иначе несуществующие URL будут с 200 и Яндекс будет ругаться.

### Vercel

В корне проекта создайте `vercel.json`:

```json
{
  "rewrites": [
    { "source": "/blog/(.*)", "destination": "/index.html" }
  ]
}
```

Главная и известные пути (/about, /services и т.д.) должны отдаваться как статические файлы (папки с `index.html`). Несуществующие пути тогда будут давать 404.

### Nginx

Для SPA с отдельными HTML по путям:

- Раздавайте статику из папки сборки.
- Для `/blog/` и вложенных путей отдавайте `index.html`:  
  `try_files $uri $uri/ /index.html =404;`  
  или отдельный `location /blog/ { try_files $uri /index.html; }`.
- Не отдавайте один и тот же `index.html` с 200 на все запросы — иначе несуществующие URL не будут 404.

## Обновление мета для новых страниц

Мета для статических страниц задаются в `src/seo/routesMeta.ts`. При добавлении нового статического маршрута добавьте в этот объект путь и соответствующие `title` и `description`.
